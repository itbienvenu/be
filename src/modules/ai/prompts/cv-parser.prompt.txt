1. Extract the following from the CV:
   - **first_name** & **last_name**: Split the full name.
   - **email**: Extract candidate email.
   - **headline**: A short professional title based on their experience.
   - **bio**: A concise professional summary.
   - **location**: City and Country.
   - **gender**: Male/Female/Other.
   - **skills**: List of tools/technologies with level and years.
   - **experience**: List of jobs with company, role, dates, work_type, and description.

   - **Required Fields**: For missing required fields, use defaults: empty strings `""` for strings, `0` for numbers, `[]` for arrays.
   - **CRITICAL - Formatted Fields**: If a field has a required format (like email or date) and the data is missing, use these placeholders:
     - **email**: Use `notfound@example.com`
     - **date** (YYYY-MM-DD): Use `1970-01-01`
   - **Optional Fields (URIs)**: If fields like `profile_picture`, `social_links`, or project `link` are not found, **OMIT them entirely** from the JSON. Do not use empty strings for URIs.
   - **IMPORTANT**: Match the Enums in the schema exactly (e.g., "Beginner", "Intermediate").

3. Output rules:
   - Only return valid JSON matching the schema exactly.
   - No extra explanations, comments, or code blocks.
   - Output **only JSON**.

Use the schema strictly.
